# Colors

The palette is built in three layers: the **brand colours** (the literal Brand Book
values), the **capability palette** (`--chart-1…5`, colour-coding the product spine), and
the **semantic shadcn tokens** that every component consumes. The semantic tokens
re-resolve per theme — flip the **theme toggle in the toolbar** and the semantic swatches
below recolour live.

## Brand colours

The official Crash Override palette (Brand Book v1). Five colours, no blue.

## Capability palette (`--chart-1…5`)

The product spine, colour-coded. Used as 3px accent bars, eyebrow labels, and chart
series — rarely as flat fills. These resolve from theme tokens, so they also track
light/dark.

## Semantic tokens (shadcn bridge)

The exact CSS variables shadcn/ui expects, mapped to the brand. Painted with
`bg-[var(--token)]` so each chip **reflects the active theme** — toggle dark/light in the
toolbar to watch them flip. `--primary` is Atomic lime on dark, Fandango purple on light.

> **Why oklch?** The semantic tokens are defined in `oklch(…)` for perceptually uniform
> lightness across both themes. You never reference them directly — you use the Tailwind
> utilities (`bg-primary`, `text-muted-foreground`, `border-border`) or the component
> props, and they resolve correctly in whichever theme is active.
